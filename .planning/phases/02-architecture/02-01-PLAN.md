<objective>
Extract the core engine into a standalone service that can run without any GUI dependency.

Purpose: The current architecture tightly couples the IDS engine to PyQt5 via IDSWorker (QObject) and MainWindow. This prevents headless operation (e.g., on servers without a display). This plan creates a clean engine layer that MainWindow can consume as a client — not as the owner.

Output: `core/engine.py` with `NetGuardEngine` class, `core/interfaces.py` with event handler protocol, `--headless` CLI flag in `main.py`, and tests proving the engine works without Qt.
</objective>

           self._running = False

           """Start the engine: psutil monitor + scheduler."""
           # Wire IDS callbacks to dispatch to handlers
           self._ids.on_ip_flagged = self._dispatch_ip_flagged
           # Start psutil monitoring thread
           # Start scheduler
